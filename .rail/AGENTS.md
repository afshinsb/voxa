# Agent Rules

These rules apply to AI coding agents working in this repository.

## Model 1 - normal workflow

```text
Create GitHub issue -> rail next --copy -> paste prompt into Codex/Claude/Cursor/Aider -> agent edits only issue scope -> rail verify --copy -> AI reviewer audit -> rail ship
```

For Codex, Claude Code, Cursor, or Aider, use the prompt generated by `rail next --copy`. Do not manually invent a new coding-agent prompt.

An AI reviewer such as ChatGPT, Claude, or another LLM is the planner/reviewer/workflow controller. The coding agent edits locally. The AI reviewer audits the result after `rail verify --copy`.

Before scoped issues exist, use `rail plan --copy` with a GitHub-connected AI agent to create a phased roadmap issue and the first small execution slice, then run `rail import`. After several shipped issues, use `rail phase --copy` for phase audit and run `rail import` again. Coding still happens one GitHub Issue at a time.

## Core rules

- Work only on the active GitHub issue or explicitly requested task.
- Keep scope small and surgical.
- Touch only necessary files.
- Do not commit.
- Do not close issues.
- Do not create unrelated roadmaps or task folders.
- Do not silently update the roadmap or create extra tasks unless the planning or phase prompt asks.
- Do not run broad/full test suites unless the issue explicitly asks for them.
- Run only focused checks related to the issue, or the exact checks requested by the user or AI Rail.
- Do not rewrite architecture unless the issue explicitly asks.
- Stop and explain if the task requires broader architecture or extra files.

## AI Rail files

Do not read or modify `.rail/rail.py`, `.rail/state/`, or `Makefile` unless the GitHub issue is specifically about AI Rail/tooling.

`.rail/PROJECT.md` is the full local project memory and roadmap brain. The GitHub roadmap issue is the remote roadmap mirror. GitHub implementation issues are only the active execution queue.

For normal app tasks, only read:

- `.rail/AGENTS.md`
- `.rail/PROJECT.md`
- `.rail/CODEX.md`
- the active GitHub issue

## Required final summary

When finished, summarize:

- files changed
- behavior changed
- focused checks run, if any
- remaining risks
