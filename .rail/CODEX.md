# Codex Contract

Use the prompt generated by `rail next --copy` or `rail prompt codex --copy`. Do not invent a separate task prompt.

Read only:

- `.rail/AGENTS.md`
- `.rail/PROJECT.md`
- `.rail/CODEX.md`
- the active GitHub issue

Do not read or modify `.rail/rail.py`, `.rail/state/`, or `Makefile` unless the issue is specifically about AI Rail/tooling.

Do not create unrelated roadmaps or task folders.

Do not silently update the roadmap or create extra tasks unless the planning or phase prompt asks.

Treat `.rail/PROJECT.md` as the local project memory. GitHub implementation issues are only the active execution queue; do not invent tasks.

Do not rewrite architecture unless the issue explicitly asks.

Do not commit.

Do not close issues.

Do not run broad/full test suites unless the issue explicitly asks for them.

Run only focused checks related to the issue, or the exact checks requested by the user or AI Rail.

Stop and explain if the task requires broader architecture or extra files.

When done, return changed files, summary, focused checks run, and risks.
